# Contributing to Rent Management System

Thank you for your interest in contributing to this project.

Before making any major changes, please create an Issue first and discuss the change with the maintainer.

## Reporting Bugs

If you find a bug:

1. Check if an issue already exists.
2. If not, create a new issue.
3. Include:

   * A clear description of the problem
   * Steps to reproduce it
   * Expected behavior
   * Actual behavior
   * Screenshots if applicable
   * Device and Android version information

Please do not immediately open a Pull Request for bug fixes without creating an Issue first.

## Suggesting Features

Feature suggestions are welcome.

Before implementing a new feature:

1. Create an Issue describing the feature.
2. Explain why the feature would be useful.
3. Wait for discussion and approval before starting work.

This helps avoid duplicate work and ensures that proposed changes align with the project's direction.

## Pull Requests

Before submitting a Pull Request:

* Ensure the project builds and compiles successfully.
* Test your changes before submitting.
* Keep changes focused and relevant.
* Use clear and descriptive commit messages.
* Link the related Issue whenever possible.

Pull Requests that introduce large changes without prior discussion may be closed.

## Code Style

Please follow the existing code style and project structure.

Consistency is preferred over personal formatting preferences.

## Questions

If you are unsure about anything, open an Issue and ask before starting work.

Thank you for contributing.
